┌────────────────────────────────────────────────────────────────────────────┐

│                                                                            │

│      └── MISS → filesystem lookup → inode load from disk                │
│                                                                            │
│  FINAL RESULT: inode found (or initialized in RAM)                       │
└────────────────────────────────────────────────────────────────────────────┘
        │
        │  inode (file metadata + block mapping)
        ▼
┌────────────────────────────────────────────────────────────────────────────┐
│                    FILE OBJECT CREATION (IMPORTANT STEP)                  │
│                                                                            │
│  struct file {                                                           │
│      inode *inode;   ← points to inode                                    │
│      f_pos = 0;      ← file offset (THIS is the pointer)                 │
│  }                                                                        │
│                                                                            │
│  fd → file*                                                               │
└────────────────────────────────────────────────────────────────────────────┘
        │
        ▼
┌────────────────────────────────────────────────────────────────────────────┐
│                          READ / WRITE PATH                                │
└────────────────────────────────────────────────────────────────────────────┘
        │
        │  read(fd, buf, N)
        ▼
┌────────────────────────────────────────────────────────────────────────────┐
│                        1. FILE OBJECT RESOLVE                            │
│                                                                            │
│   fd → struct file                                                        │
│        ├── f_pos (offset)                                                │
│        └── inode (metadata + block map)                                  │
└────────────────────────────────────────────────────────────────────────────┘
        │
        ▼
┌────────────────────────────────────────────────────────────────────────────┐
│                        2. PAGE CACHE CHECK                               │
│                                                                            │
│   ├── HIT  → return data (NO disk I/O)                                   │
│   └── MISS → go to filesystem                                             │
└────────────────────────────────────────────────────────────────────────────┘
        │
        ▼
┌────────────────────────────────────────────────────────────────────────────┐
│                     3. FILESYSTEM MAPPING (CORE LOGIC)                   │
│                                                                            │
│   offset → logical block                                                  │
│   logical block → physical block                                          │
│                                                                            │
│   (via inode + extents)                                                  │
└────────────────────────────────────────────────────────────────────────────┘
        │
        ▼
┌────────────────────────────────────────────────────────────────────────────┐

└────────────────────────────────────────────────────────────────────────────┘
        │
        ▼
┌────────────────────────────────────────────────────────────────────────────┐

        │
        ▼
┌────────────────────────────────────────────────────────────────────────────┐
│                    6. STORAGE DEVICE / EBS                                │

│   ├── Local SSD/NVMe OR                                                   │

        │
